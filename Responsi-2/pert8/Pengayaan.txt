Setelah anda menyelesaikan kode, tolong jawab pertanyaan dibawah ini ya =


1. Apa tujuan utama adanya interface dalam pemrograman berorientasi objek, dan bagaimana interface membantu
menciptakan kontrak perilaku?
2. Mengapa interface dapat memiliki banyak implementasi, dan apa makna konseptual dari kemampuan tersebut dalam
desain sistem?
3. Apa perbedaan mendasar antara extends dan implements dalam konteks interface, jika dilihat secara konsep tanpa
melihat contoh kode?
4. Apa alasan diperkenalkannya default method dalam interface, dan bagaimana fitur ini memengaruhi keberlangsungan
(compatibility) suatu sistem yang sudah berjalan?
5. Bagaimana konsep multiple interface memperluas fleksibilitas desain tanpa menimbulkan masalah kompleksitas yang
biasa muncul pada multiple inheritance?

Jawaban:

1. Interface digunakan untuk menentukan standar perilaku yang harus dimiliki oleh suatu class, sehingga interface
berfungsi sebagai kontrak yang memastikan class yang mengimplementasikannya menyediakan method sesuai aturan.
Dengan adanya interface, sistem dapat memaksa keseragaman perilaku meskipun objek yang dibuat berbeda jenisnya.

2. Interface dapat memiliki banyak implementasi karena hanya mendefinisikan struktur method tanpa menentukan logikanya,
sehingga setiap class bebas membuat versinya sendiri. Secara konsep, hal ini memberikan variasi perilaku namun tetap
dalam satu aturan yang sama, sehingga sistem lebih fleksibel dan efisien

3. Extends berarti mewarisi sifat atau perilaku yang sudah dimiliki class atau interface lain, sedangkan implements
berarti class berjanji untuk menyediakan implementasi dari method di dalam interface. Jadi extends adalah pewarisan
kemampuan yang ada, sedangkan implements adalah pemenuhan kontrak

4. Default method diperkenalkan supaya ketika interface diperbarui, class lama tetap dapat berjalan tanpa harus
langsung melakukan perubahan. Fitur ini membantu menjaga kompatibilitas sistem, terutama pada sistem besar yang sudah
digunakan banyak bagian aplikasi

5. Multiple interface memberi fleksibilitas karena class dapat memiliki banyak kemampuan sekaligus tanpa menimbulkan
konflik seperti pada multiple inheritance. Hal ini terjadi karena interface hanya membawa definisi method, bukan
implementasi atau data, sehingga tidak menimbulkan benturan logika
